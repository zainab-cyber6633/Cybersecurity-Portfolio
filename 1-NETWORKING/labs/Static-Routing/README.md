Static Routing Lab

Author

Zainab Ijaz

Overview

This lab demonstrates the configuration of Static Routing using two Cisco routers. Static routes were manually configured to enable communication between two different LAN networks.

Objectives

- Configure IP addresses on router interfaces.
- Connect two LANs using two routers.
- Configure static routes on both routers.
- Verify end-to-end connectivity using ping.
- Save the router configuration.

Topology

- 2 Cisco Routers (R1, R2)
- 2 Cisco Switches (S1, S2)
- 2 PCs (PC1, PC2)



Configuration Summary

- Configured router interfaces with IP addresses.
- Enabled interfaces using "no shutdown".
- Configured static routes on both routers.
- Configured default gateways on both PCs.
- Saved the running configuration.

Verification

- "show ip interface brief"
- "show ip route"
- "show running-config"
- "ping"
- Successful communication between PC1 and PC2.

Skills Demonstrated

- Static Routing
- Router Configuration
- IP Addressing
- Cisco IOS Commands
- Network Troubleshooting
- Connectivity Verification

Tools Used

- Cisco Packet Tracer
- Cisco IOS CLI

Outcome

Static routing was successfully configured, allowing devices on different networks to communicate through manually configured routes.
